# Git Skills Tracker

Track your progress as you learn Git! Check off each skill as you master it.

## Basic Git Skills

- [ ] Install and configure Git
- [ ] Set up Git identity (`user.name` and `user.email`)
- [ ] Initialize a repository (`git init`)
- [ ] Check repository status (`git status`)
- [ ] Stage files (`git add`)
- [ ] Create commits (`git commit`)
- [ ] View commit history (`git log`)
- [ ] View file differences (`git diff`)

## Branching Skills

- [ ] Create a new branch (`git branch`)
- [ ] Switch between branches (`git checkout` or `git switch`)
- [ ] Create and switch to a branch in one command (`git checkout -b`)
- [ ] List all branches (`git branch -a`)
- [ ] Merge branches (`git merge`)
- [ ] Delete branches (`git branch -d`)

## Collaboration Skills

- [ ] Clone a repository (`git clone`)
- [ ] Add remote repositories (`git remote add`)
- [ ] Fetch changes from remote (`git fetch`)
- [ ] Pull changes from remote (`git pull`)
- [ ] Push changes to remote (`git push`)
- [ ] Create pull requests
- [ ] Review code changes
- [ ] Resolve merge conflicts

## Advanced Skills

- [ ] Undo changes (`git restore`, `git reset`)
- [ ] Rewrite history (`git rebase`)
- [ ] Cherry-pick commits (`git cherry-pick`)
- [ ] Stash changes (`git stash`)
- [ ] Tag releases (`git tag`)
- [ ] Use `.gitignore` effectively
- [ ] Configure Git aliases
- [ ] Use `git blame` to track changes

## Best Practices Learned

- [ ] Write clear, descriptive commit messages
- [ ] Commit often with logical chunks of work
- [ ] Keep branches focused on single features
- [ ] Pull before pushing to avoid conflicts
- [ ] Review changes before committing
- [ ] Use branches for experimentation
- [ ] Never commit sensitive data (passwords, API keys)
- [ ] Keep repository history clean and organized
